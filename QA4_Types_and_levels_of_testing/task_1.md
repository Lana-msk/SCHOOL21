### **Классификация тестирования**

| По стадии проведения процесса тестирования | По знанию устройства системы | По направленности тестового сценария | По степени автоматизации | По выполнению кода |
|--------------------------------------------|-----------------------------|--------------------------------------|--------------------------|---------------------|
| **Alfa-тестирование**<br><br>Удобство и логика макета интерфейса силами дизайнера | **Black-Box (Черный ящик)**<br><br>Функционал экрана перевода без доступа к коду | **Позитивное тестирование**<br><br>Стандартный успешный сценарий перевода с корректными данными | **Ручное тестирование**<br><br>Exploratory-тестирование первой сборки | **Статическое тестирование**<br><br>Анализ интерфейса без запуска (Figma)<br><br>Код-ревью |
| **Beta-тестирование (внутреннее)**<br><br>Стабильность и удобство приложения для внутренних сотрудников | **Grey-Box (Серый ящик)**<br><br>Интеграция модуля перевода с тестовой БД | **Негативное тестирование**<br><br>Ввод невалидных данных, проверка сообщений об ошибке | **Автоматизированное тестирование**<br><br>Критические регрессионные сценарии (скрипт на Java) | **Динамическое тестирование**<br><br>Функциональное тестирование с запущенным приложением |
| **Приемочное тестирование (UAT)**<br><br>Соответствие приложения требованиям перед передачей заказчику | **White-Box (Белый ящик)**<br><br>Анализ покрытия кода модуля перевода | **Тестирование безопасности**<br><br>Базовая проверка уязвимостей, шифрование данных | **Полуавтоматизированное тестирование**<br><br>Автогенерация данных + ручной анализ | **Динамическое тестирование (нагрузочное)**<br><br>Нагрузочное тестирование API |
